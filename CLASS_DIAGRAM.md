## ğŸ”— KEY RELATIONSHIPS

1. **Controllers â†’ Service Interfaces**
   - Controllers depend on interfaces, not implementations
   - Promotes loose coupling

2. **Service Implementations â†’ Repositories**
   - Service layer orchestrates repository calls
   - Repositories provide data access abstraction

3. **Repositories â†’ Spring Data JPA**
   - All repositories extend `JpaRepository<Entity, ID>`
   - Spring generates implementation at runtime

4. **Entities â†” Database**
   - JPA/Hibernate maps entities to tables
   - Bidirectional relationships managed

5. **Strategy Pattern (Pricing)**
   - `ParkingServiceImpl` depends on `PricingStrategy` interface
   - Can swap implementations without changing service code
